> **Unassigned Devices est appelé UD.**
>
> Passez votre souris sur une zone active de la page UD et une info-bulle vous montrera ce qu'il faut faire en cliquant sur cette zone.
> Vous pouvez monter des périphériques USB, des lecteurs SATA, des partages SMB/NFS distants et des fichiers ISO avec UD.  Tous les périphériques dont l'option de montage automatique est activée seront montés au démarrage de la baie.  Tous les lecteurs et montages SMB/NFS sont démontés lorsque la matrice est arrêtée par l'événement 'stopping_svcs'.
> Si vous souhaitez partager votre lecteur, vous pouvez activer le commutateur de partage.  La valeur par défaut des partages SMB distants est Accès public en lecture/écriture.  Activez la sécurité SMB par utilisateur dans les paramètres de Unassigned Devices.  Les partages NFS sont exportés et l'accès est en lecture/écriture.  L'exportation de périphériques NFS est activée dans les paramètres de Unassigned Devices. Vous pouvez également activer un script commun qui sera exécuté à chaque montage de disque.  Pour partager un périphérique UD, le partage doit être activé dans les paramètres UD et le commutateur doit être activé pour partager le périphérique en question.  Les partages de suppression SMB/NFS et les montages ISO sont toujours partagés.
> Après avoir saisi un point de montage, appuyez sur &lt;Entrée&gt; pour enregistrer la modification.
>
> Des options supplémentaires sont disponibles lorsque vous cliquez sur l'icône <i style='color:black;font-weight:bold;' class="fa fa-plus-square"></i> par identification du périphérique.  Lorsque le disque est démonté, le point de montage du périphérique peut être modifié.  L'étiquette du disque sera également modifiée.  Un disque peut être formaté et un fsck exécuté pour vérifier le disque.
>
> Les paramètres de sécurité "Réglages->Unassigned Devices" pour SMB et NFS doivent être correctement définis pour que les partages SMB et NFS fonctionnent correctement.
>
> Si le bouton de montage est grisé pour un partage distant SMB/NFS, le serveur distant ne répond peut-être pas au ping.  Le serveur du partage distant doit répondre à un ping sinon UD le considérera comme hors ligne.
>
> UD dispose d'un mode destructif qui permet de supprimer des partitions de disque et de formater des disques.	Si le mode destructif n'est pas activé dans les paramètres de l'UD, vous NE POURREZ PAS formater un disque ou supprimer des partitions.  Allez dans 'Réglages->Unassigned Devices' pour définir le mode destructif.
>
> **Pour formater un disque :**
>
> + Le mode destructif doit être activé.
> + Le disque ne doit plus avoir de partition. Démontez le disque, cliquez sur l'icône <i style='color:black;font-weight:bold;' class='fa fa-plus-square'></i> et cliquez sur toutes les icônes <i style='color:red;font-weight:bold;' class='fa fa-remove hdd'></i> pour supprimer les partitions.
> + Si le disque a fait l'objet d'un pré-formatage et affiche un bouton 'Formater' grisé, cliquez sur l'icône <i style='color:black;font-weight:bold;' class='fa fa-plus-square'></i> puis cliquez sur l'icône <i style='color:red;font-weight:bold;' class='fa fa-remove hdd'></i> pour supprimer le fichier d'état de pré-formatage.
>
> **Remarque : Un disque partitionné en UD est compatible avec le partitionnement du disque de la matrice et peut être ajouté à la matrice.  Les formats de fichiers pris en charge sont XFS, XFS chiffré, BTRFS et BTRFS chiffré.**
>
> **Disque chiffrés :**
>
> + Tout disque formaté en UD qui est chiffré utilisera le mot de passe/passphrase de la matrice et pourra être incorporé dans la matrice.  Vous pouvez également définir un mot de passe unique lors de son formatage.  Si vous définissez un mot de passe sur le disque lorsqu'il est formaté, vous devrez entrer un mot de passe de disque dans Unassigned Devices->Règlages pour ce disque.
> + Tout disque qui n'a pas été chiffré avec le mot de passe/passphrase de la matrice peut être monté en définissant un mot de passe par disque dans Unassigned Devices->Réglages.
>
> **Script Unassigned devices :**
>
> UD inclut un script qui est utilisé pour monter et démonter les périphériques et les montages SMB/NFS. Ces scripts peuvent être utilisés à partir de la ligne de commande ou dans vos propres scripts si nécessaire.
>
> + '/usr/local/sbin/rc.unassigned mount autodevices' - tous les périphériques définis pour le montage automatique seront montés.
> + '/usr/local/sbin/rc.unassigned mount autoshares' - tous les montages SMB/NFS définis pour le montage automatique seront montés.
> + '/usr/local/sbin/rc.unassigned umount auto' - tous les périphériques et les montages SMB/NFS définis pour le montage automatique seront démontés.
> + '/usr/local/sbin/rc.unassigned umount all' - tous les périphériques et les montages SMB/NFS sont démontés en vue de l'arrêt de la matrice.
> + '/usr/local/sbin/rc.unassigned mount /dev/sdX or devX' - monte le périphérique sdX où 'X' est l'indicatif du périphérique.  devX est le nom du périphérique dans la page UD.  Si le nom du périphérique est 'Dev 1', utilisez dev1 comme périphérique à monter.
> + '/usr/local/sbin/rc.unassigned umount /dev/sdX or devX' - démonte le périphérique sdX où 'X' est l'indicatif du périphérique. devX est le nom du périphérique dans la page UD.  Si le nom du périphérique est 'Dev 1', utilisez dev1 comme périphérique à démonter. Vous pouvez utiliser cette commande dans un script UD pour démonter le périphérique une fois le script terminé.
> + '/usr/local/sbin/rc.unassigned spindown devX' - arrête la rotation d'un disque.  Le devX est le nom du périphérique dans la page UD.  Si le nom du périphérique est 'Dev 1', utilisez dev1 comme périphérique à mettre hors tension.  Les SSD ne seront pas mis hors tension.
> + '/usr/local/sbin/rc.unassigned mount SOURCE' - où SOURCE est la source SMB/NFS.
> + '/usr/local/sbin/rc.unassigned umount SOURCE' - où SOURCE est la source SMB/NFS.
>
> Soyez prudent en utilisant la désignation /dev/sdX car elle peut changer après un redémarrage.
